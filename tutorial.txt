Ini adalah tutorial langkah per langkah untuk pembuatan server nest-task dalam satuan commit. 
Itu artinya pantau commit nya pada resipotery untuk melihat langsung perubahaanya sesuai dengan tutorial.txt

1. Membuat file module dengan Nest-CLI:
  - Hapus semua file dalam source kecuali main.ts lalu ubah main.ts (karena akan memulai dari awal)
  - Jalankan pada terminal `nest generate module tasks` untuk membuat folder dan module tasks (untuk seterusnya akan sering menggunakan nest-cli)
  - Lalu gantikan AppModule pada main.ts dengan TasksModule

2. Membuat controller dengan Nest-CLI:
  - Jalankan `nest generate controller tasks/tasks --flat` (task/task berarti directory nya, flat artinya tidak membuat folder baru untuk controller dalam folder tasks, maksudnya setara dengan tasks.module)
  - nest-cli otomatis membuat controller, unit testing, beserta import controller pada module
  
3. Mendefinisikan route di controller:
  - Ubah tasks.controller.ts dengan tiga route, menampilkan seluruh tasks (get), membuat task (post), menampilkan task (get)
  - Cara ini lebih efektif (menambahkan decorator 'tasks' di controller) supaya tidak mengetik /tasks terus pada method nya

4. Mendapatkan data yang dikirim dalam request:
  - ubah tasks.controller untuk menambahkan Body dan Param dari nestjs/common.

5. Menambahkan data validator dengan validasi pipe:
  - ubah main.ts untuk menambahkan global validation (hanya untuk contoh validasi), yg artinya tiap kali request terdapat validasi
  - install class-validator dan class-transformer dengan `yarn add class-validator class-transformer`
  - buat file dengan nama /dtos/create-task.dto.ts di folder tasks lalu tambahkan validasi seperti contoh.
  - terapkan validator yang dibuat di module untuk method Post sesuai contoh
  - untuk penyesuaian eslint `eslint.config.mjs`, tambahkan override khusus jika cukup menganggu
